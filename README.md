## Bitonic Sorter – Embedded/FPGA Project

This repository contains a hardware implementation of a **bitonic sorting network** for the **Intel Cyclone V** FPGA on the **DE1‑SoC** board.  
It is an embedded/FPGA course project implemented in **SystemVerilog** using **Quartus Prime Lite** and **ModelSim‑Altera**.

### Features

- **Bitonic sorting network in hardware**
  - Main scalable sorter (`Bitonic_Sort4`) with parameterizable data width and element count.
  - Alternative 8‑element sorter (`Bitonic_Sort2`) using a simple FSM for comparison.
- **FPGA target**
  - Family: **Cyclone V**
  - Device: `5CSEMA5F31C6`
  - Board: **DE1‑SoC**
- **Tool flow**
  - Designed and compiled with **Quartus Prime Lite 18.1**.
  - Simulated with **ModelSim‑Altera (SystemVerilog)**.

### How the design works (high level)

- The sorter operates on a packed input bus of `ELEMENTS * BITWIDTH` bits.
- Internally, the design uses a **bitonic sorting network**, a fixed sequence of compare‑and‑swap stages:
  - `Bitonic_Sort4` processes the input in chunks of 8 elements, applying multiple steps of compare‑swap operations and iterating across the full vector until all stages are complete.
  - `Bitonic_Sort2` shows a smaller, 8‑element network driven by a finite state machine.
- The top‑level module `top_level_test` connects clock, reset, enable, input vector, output vector, and a `done` signal for easy integration and testing on the DE1‑SoC.

### Building with Quartus

1. Open **Quartus Prime Lite 18.1** (or compatible).
2. `File → Open Project...` and select `bitonic_sorter_25.qsf`.
3. Confirm:
   - Device family: **Cyclone V**
   - Device: `5CSEMA5F31C6`
   - Top‑level entity: `top_level_test`
4. Run **Processing → Start Compilation**.
5. The compiled bitstream (`.sof`) will be generated in `output_files/`.

### Programming the DE1‑SoC

1. Connect the DE1‑SoC board via USB‑Blaster.
2. In Quartus, open **Tools → Programmer**.
3. Add the generated `.sof` file from `output_files/`.
4. Select the Cyclone V device and click **Start** to program the FPGA.

### Simulation

- RTL simulations can be run with **ModelSim‑Altera** using:
  - Quartus’ integrated flow (`Tools → Run Simulation Tool → RTL Simulation`), or
  - The generated scripts and waveforms under `simulation/` and `Waveform*.vwf`.
- You can modify the waveform files or create a dedicated SystemVerilog testbench to apply custom stimuli.
